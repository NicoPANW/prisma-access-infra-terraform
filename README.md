# SCM Prisma Access Terraform Automation

This repository contains the Terraform configurations and helper Python scripts to automate deployment of Remote Networks (RN) and Service Connections (SC) on Strata Cloud Manager. It has been tested on a freshly activat new tenant.

## Purpose of each file

### 1. Static Project Files (Already Created)
These are the core engineering files that define your infrastructure schemas, variable parameters, and operational logic. They should be tracked in Git.

*   **`main.tf`**: The primary Terraform configuration file. It contains the SCM provider initialization, custom cryptographic profiles, resource declarations for Service Connections and Remote Networks, dynamic bandwidth calculation logic, and security rules.
*   **`variables.tf`**: Defines the strict type schemas, validations, and structures for your Terraform input variables. It contains no hardcoded values to preserve portability.
*   **`terraform.tfvars`**: Your environment-specific input values file. This is where you specify the human-readable display region names, subnets, PSKs, and bandwidth parameters for your deployment.
*   **`token_cache_service.py`**: An atomic utility script designed to run either independently or as a pre-plan step inside Terraform. It manages the OAuth2 client credentials grant flow, handles file locking, and caches JWTs.
*   **`RN_fetch_cloud_mapping_regions.py`**: A helper script that fetches the locations database globally from SCM and formats it into a clean translation map for Terraform's location lookups.
*   **`scm_commit_push.py`**: An operations script used to push and commit staged candidate configurations directly to the SCM active plane after validation.

---

### 2. Dynamically Auto-Generated Files
These files are automatically generated or updated on your local disk at runtime by your scripts and Terraform. They are excluded from Git via `.gitignore`.

*   **`jwt-token.json`**: Generated and refreshed atomically by `token_cache_service.py`. It holds the active JWT token, host gateway target, and expiration metadata required by the SCM Terraform provider.
*   **`RN-region-mappings.json`**: Generated by `RN_fetch_cloud_mapping_regions.py`. It contains the raw, unmodified locations array directly from SCM's locations database.

---

### 3. User-Provided Configuration Files
This file is the single source of secret credentials for the workspace and must be supplied manually on new clones.

*   **`auth-token.json`**: Must be created manually in the root folder using your client credentials and TSG ID scope. It acts as the static, read-only credential source for all automation scripts.


## Workflow

1. Clone repo
   ```bash
   git clone https://github.com/NicoPANW/prisma-access-infra-terraform
   cd prisma-access-infra-terraform
   ```

2. Create in SCM a service-account for the desired TSG. Copy your credentials file into the root of this project as `auth-token.json`. It must follow this structure:
   ```json
   {
     "client_id": "YOUR_CLIENT_ID",
     "client_secret": "YOUR_CLIENT_SECRET",
     "host": "api.strata.paloaltonetworks.com",
     "protocol": "https",
     "scope": "tsg_id:YOUR_TSG_ID"
   }
   ```
3. Fetch the current SCM cloud compute region map file before planning. It will generate a file RN-region-mappings.json:
   ```bash
   python3 RN_fetch_cloud_mapping_regions.py
   ```
4. Review and update as need be terraform.tfvars file. Note for region parameter for both SC and RN you MUST use the region dispaly name (ie France North, Australia South, Sweden). You can know this via reading the file RN-region-mappings.json
   
5. Initialise Terraform and apply.
   ```bash
   terraform init
   terraform apply
   ```

6. You can check the result in SCM UI, then you can commit config with this script
   ```bash
   python3 scm_commit_push.py
   ```
7. Destroy all ressources created and commit.
   ```bash
   terraform destroy
   python3 scm_commit_push.py
   ```
